# Security Policy

## Supported Versions

We actively monitor and patch security vulnerabilities for the following versions of CreatorBharat:

| Version | Supported |
| :--- | :--- |
| 1.0.x |  Yes |
| < 1.0.0 | ❌ No |

---

## Reporting a Vulnerability

We take the security of CreatorBharat, our creator nodes, and brand escrow accounts extremely seriously. If you find a security vulnerability, please **do not open a public issue**. Instead, report it privately to ensure our users' data remains secure.

Please report all security issues directly to the lead architect:
- **Contact Email:** mohmmad.dilshan.business@gmail.com (or your corporate security email)

Please include the following details in your report:
- Type of vulnerability (e.g., XSS, CSRF, SQL Injection, Auth Bypass).
- Step-by-step instructions or proof-of-concept (PoC) code to reproduce the issue.
- Potential impact (how it affects users, transactions, or server nodes).

---

## Our Response Process

1. **Acknowledgment:** We will acknowledge receipt of your vulnerability report within **48 hours**.
2. **Evaluation:** Our trust & security team will evaluate the impact and reproduce the vulnerability.
3. **Patching:** If verified, we will develop a patch and apply it to our production nodes.
4. **Disclosure:** Once the patch is deployed, we will coordinate public disclosure (if applicable) and credit the researcher.

Thank you for helping keep CreatorBharat secure!
