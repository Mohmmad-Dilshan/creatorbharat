# 🇮🇳 CreatorBharat V3 — Folder-by-Folder SaaS Implementation Blueprint

This guide maps out the exact additions, files, and integrations required in **each directory** (Frontend, Backend, and Admin Panel) to achieve a production-grade, commercially launchable SaaS platform.

---

## 💻 1. Frontend Client Requirements (`creator-bharat-v3/`)

To turn the current user interface into a dynamic client talking to a live backend database:

### A. Authentication & Session Flow Integration
* **Status Now**: UI forms exist, but validation and error states use simulation fallbacks.
* **Next Code Changes**:
  * **Forms Refactoring**: Update `/login`, `/apply` (creator signup), `/brand-register` (brand signup), and `/forgot-password` to catch backend error payloads (e.g. `400 Bad Request: Handle already taken` or `401 Unauthorized: Invalid password`) and show native saffron toast notices.
  * **Session Check**: Intercept application boot in `App.jsx` to verify if JWT session token exists in cookies/storage and fetch `/api/auth/me` to populate global Context `st.user`.

### B. Profile Builder & Onboarding Connect
* **Status Now**: Stepper checklist reads from `localStorage`.
* **Next Code Changes**:
  * **API Saves**: Modify save triggers in [ProfileBuilderPage.jsx](file:///d:/creatorbharat-1/creator-bharat-v3/src/pages/creator/ProfileBuilderPage.jsx) (Identity tab, Story tab, Deliverables tab) to fire `PUT /api/creators/me` instead of writing to local storage.
  * **Media Uploads Component**: Integrate a React image upload selector (using `react-dropzone` or simple file inputs) next to the Profile Avatar, Cover Banner, and Verification Documents sections. Submit files using multipart `FormData` to `/api/uploads/image`.

### C. Live Feeds & Discovery Search
* **Status Now**: Creator list and Opportunities lists use static seed data arrays.
* **Next Code Changes**:
  * **Dynamic Query Mapping**: Update [CreatorsPage.jsx](file:///d:/creatorbharat-1/creator-bharat-v3/src/pages/brand/CreatorsPage.jsx) (Brand creator directory) to trigger `fetchCreators` with URL query strings (e.g. `?niche=beauty&platform=instagram&state=Rajasthan`) when filter buttons are toggled.
  * **Active Campaign Application**: Update the campaign feed to query active campaign cards from `/api/campaigns` and link the "Apply Pitch" submit button to call `POST /api/campaigns/:id/apply`.

### D. Payments & Chat Client
* **Status Now**: Razorpay triggers are mocked in pricing card click events.
* **Next Code Changes**:
  * **Razorpay Web SDK**: Inject Razorpay checkout JS script. Pass the `orderId` generated by the backend API into Razorpay options and open checkout popup. Hook success callback to call `/api/payments/verify`.
  * **Socket.io Chat**: Hook a socket client in `MessagesPage.jsx` to dynamically send and receive real-time messages when talking to a brand.

## ⚙️ 2. Backend API Requirements (`creatorbharat-backend/`)

To transition the backend from a basic REST routing template into an enterprise-grade SaaS database manager:

### A. Controllers & Middleware Separation
* **Next Code Changes**:
  * Create `src/controllers/` folder. Extract heavy SQL queries and Zod validations out of the routes directory into clean controller files (e.g. `authController.js`, `creatorController.js`).
  * Create `src/utils/uploader.js`: Configure `multer` disk/memory storage and Cloudinary integration parameters.
  * Create `src/utils/mailer.js`: Helper function to configure Resend API credentials and send HTML formatted transaction notifications.

### B. Missing Endpoints & Webhooks
* **Next Code Changes**:
  * **File Upload Endpoints (`routes/uploads.js`)**: Add `/api/uploads/image` (Multer captures file -> Cloudinary uploads -> returns secure CDN URL).
  * **Razorpay Asynchronous Webhooks**: Add `/api/payments/webhook` to listen to direct events from Razorpay (e.g. `payment.captured` or `subscription.activated`) to update PostgreSQL payment states even if the user closes their browser during checkout.
  * **Email Trigger Integration**: Add Resend client dispatches inside signup, verification request, and application pitch routers.

## 📊 3. Admin Panel Control Dashboard Requirements (`creatorbharat-admin/`)

To expand the simple admin dashboard template into a full SaaS control center:

### A. Extended Approvals Queue & User Management
* **Next Code Changes**:
  * **Details Modal Popup**: When clicking a creator in the verification queue, open a modal displaying their uploaded PAN/Aadhaar card images, connected social handles, and verification parameters before confirming the verified status.
  * **Access Controls (Bans/Suspensions)**: Add a button next to creator and brand rows in the directory tables to allow administrators to suspend users. This sets a `status` field in the database User table, preventing the suspended user from logging in.

### B. Visual Graphing & Analytics Hub
* **Next Code Changes**:
  * **Chart Library Setup**: Install charting packages like `recharts` or `chart.js`.
  * **Graph Views**: Add dynamic line charts showing daily user signups, transaction payout bar charts, and category pie charts in the Overview dashboard.

### C. Dispute & Audit Console
* **Next Code Changes**:
  * **Dispute Resolution Sheet**: Create an interface where administrators can view contested campaign escrows and choose to release funds to the creator or refund them to the brand.
  * **System Audit Logs**: Track administrator actions (e.g. which admin verified which creator, when a brand was suspended) in a dedicated database table and display in a console log stream.
